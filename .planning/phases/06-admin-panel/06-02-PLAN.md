Structure:
- POST handler at top: session_start(), load config.php, compare password (plaintext OR bcrypt — see dual-format pattern in interfaces), set $_SESSION['admin'] = true + redirect to /admin/ on success, or set $error on failure.

  <name>Task 2: admin/index.php — trip dashboard with stats, table, DnD reorder, trash</name>
  <files>admin/index.php</files>
  <action>
Create admin/index.php. Session auth guard at top (see interfaces pattern).

**Page structure** (self-contained HTML, no public header/footer):
- Load admin.css, Inter font, Font Awesome 6.
- Admin nav bar: "Viaggia col Baffo" site name (left), nav links: "Pannello" (active), "Impostazioni" → /admin/settings.php, "Tag" → /admin/tags.php, "Destinazioni" → /admin/destinations.php, "Vai al sito" (→ / , target _blank), "Esci" logout button (POST to ?action=logout).
- Logout handler: if $_GET['action'] === 'logout' or $_POST['action'] === 'logout': session_destroy(), redirect to /admin/login.php.

**PHP data layer** (at top, after session guard):
- Load all trips with load_trips().
- Separate into: $active_trips (deleted !== true), $trash_trips (deleted === true).
- Sort $active_trips by 'position' field (use usort; treat missing position as 999).
- Compute: $total = count($active_trips), $published = count(array_filter(..., published===true)), $draft = $total - $published.

**AJAX handlers** (check $_POST['action'] at top, return JSON, exit):
- action=toggle_published: find trip by slug, flip published bool, save_trips(), return {"success":true,"published":bool}.
- action=soft_delete: find trip by slug, set deleted=true, save_trips(), return {"success":true}.
- action=restore: find trip by slug, set deleted=false, save_trips(), return {"success":true}.
- action=empty_trash: filter out all trips where deleted===true (hard delete), save_trips(), return {"success":true}.
- action=reorder: accept JSON body with array of slugs in new order, update position index for each trip, save_trips(), return {"success":true}.

**Stats bar** (3 cards): "Viaggi Totali" + count, "Pubblicati" + count, "Bozze" + count.

**"Crea Nuovo Viaggio" gold button** → /admin/edit-trip.php?new=1 (top right, prominent).

**Active trips table**:
- Columns: (drag handle), Title, Continent, Tags (gold chip pills), Status (colored pill: Pubblicato=green, Bozza=gray), Actions.
- Each row: draggable="true", data-slug attribute.
- Actions per row: "Modifica" → /admin/edit-trip.php?slug={slug}, "Anteprima" → /viaggio/{slug}?preview={preview_token} (target _blank), "Elimina" button (triggers JS modal — if trip is published, show "È pubblicato. Prima renderlo bozza?" modal; if draft, show "Andrà nel Cestino." confirmation modal).
- Published toggle: clickable pill/switch next to status (AJAX action=toggle_published).
- Drag-and-drop reorder using HTML5 drag API (dragstart/dragover/drop on tbody rows). On drop, call renumberRows() then debounce-POST action=reorder with new slug order as JSON. See Research Pattern 6 for the JS implementation.

**Cestino section** (collapsible or shown below table):
- Shows only if count($trash_trips) > 0.
- Table with: Title, "Ripristina" button (AJAX action=restore), "Elimina definitivamente" button (AJAX action=hard_delete — NOT needed per CONTEXT.md — only Svuota Cestino bulk-deletes).
- "Svuota Cestino" red button: confirmation prompt then AJAX action=empty_trash.

All AJAX actions show a brief toast notification on success ("Salvato", "Eliminato", etc.) — simple fixed-position div that auto-hides after 2 seconds.

Note: The 'deleted' field may not exist on current trips.json entries — treat missing 'deleted' as false.
