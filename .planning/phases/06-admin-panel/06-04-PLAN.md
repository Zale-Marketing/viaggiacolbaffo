Create admin/settings.php. Session auth guard at top.

**POST handler:**
1. Read existing admin-config.json (or start with empty array if not exists).
2. Merge submitted form values into the array.
3. Password handling: if $_POST['admin_password'] is non-empty, hash with password_hash($_POST['admin_password'], PASSWORD_DEFAULT) and store in 'admin_password'. If empty string submitted, keep existing password (do not overwrite with empty).
4. Write updated array back to DATA_DIR . 'admin-config.json' using flock pattern (same as save_trips). Use JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES.
5. Redirect to ?saved=1 after successful save.

**HTML form sections** (grouped in cards):

Section 1 — "Sicurezza":
- Admin password: password input with placeholder "Lascia vuoto per non modificare". Note: "La password attuale è quella in config.php finché non ne imposti una qui."
- No username field (single-password scheme).

Section 2 — "Contenuto Sito" (urgency bar + company data):
- Urgency bar text: text input (label: "Testo barra urgenza — mostrato sotto l'hero in homepage")
- Company name: text input
- P.IVA: text input
- Company address: text input

Section 3 — "Webhook URL":
- Default webhook (preventivo): text input

Section 4 — "WhatsApp e Form":
- WhatsApp number: text input (with placeholder e.g. "+39 333 1234567")

Section 5 — "AI Form Generator":
- Anthropic API key: password-type input (type="password") so the key is not visible as plain text. Note: "Usato per la generazione automatica del Form Config con Claude AI."

Each section is a white card. Save button "Salva Impostazioni" (gold primary) at bottom.
Pre-populate all inputs from the current admin-config.json values (loaded at top of PHP).

Show "Impostazioni salvate." success banner if ?saved=1 in URL.

**admin/tags.php:**

Session auth guard at top.

POST AJAX handlers (check $_POST['action'], return JSON, exit):
- action=add_tag: read name, slug (auto-generated from name if empty: strtolower + trim + spaces to hyphens), category. Validate slug uniqueness. Append to load_tags() array, save_tags(). Return {"success":true, "tag": {...}}.
- action=delete_tag: read slug. Remove from tags array, save_tags(). THEN load all trips, filter tags array on each trip removing this slug, save_trips(). Return {"success":true}.

HTML page:
- Admin nav (same pattern as other pages).
- "Aggiungi Tag" card at top: text input for name, select for category (continente/tipo viaggio/per chi/mese/altro), "Aggiungi" button → AJAX action=add_tag → appends new chip to the correct group on success.
- Tags displayed grouped by category. Each group is a labeled section with chips. Each chip has an "×" delete button → AJAX action=delete_tag with confirmation: "Elimina questo tag da tutti i viaggi?" → on confirm, removes chip from DOM.
- Show category label headers: "Continente", "Tipo Viaggio", "Per Chi", "Mese", "Altro" (fallback for tags without recognized category).

**admin/destinations.php:**

Session auth guard at top.

This page has two modes:
1. List mode (no ?slug=): show 6 destination cards as a grid with the destination name, hero image thumbnail, and "Modifica" button.
2. Edit mode (?slug=america etc.): show full edit form for that destination.

**List mode HTML:**
- 6 grid cards (one per slug). Each card: destination name, small hero thumbnail (img with the current hero_image URL), "Modifica" → ?slug={slug}.

**Edit mode:**

POST handler at top (if mode=edit and POST):
1. Load destinations array.
2. Update the $slug entry:
   - hero_image from $_POST['hero_image']
   - intro_paragraphs: [$_POST['intro_1'], $_POST['intro_2'], $_POST['intro_3']]
   - practical_info: array of 5 objects rebuilt from $_POST['pinfo_icon[]'], $_POST['pinfo_label[]'], $_POST['pinfo_value[]']
   - see_also: array of 4 objects from $_POST['see_name[]'], $_POST['see_image[]'], $_POST['see_desc[]']
   - curiosita: array of 3 objects from $_POST['cur_icon[]'], $_POST['cur_title[]'], $_POST['cur_text[]']
   - name field: keep existing (6 fixed slugs — name does not change).
3. save_destinations($destinations).
4. Redirect to ?slug={slug}&saved=1.

Edit form HTML (for ?slug=):
- Back link to list mode (← Tutte le Destinazioni).
- Heading: "Modifica Destinazione — {name}"
- Hero image URL input with live thumbnail preview (same JS pattern as edit-trip.php).
- "Testi introduttivi": 3 textarea fields (intro_1, intro_2, intro_3) — one per intro paragraph.
- "Informazioni pratiche": 5 rows each with icon (text input, e.g. "fa-solid fa-coins"), label input, value input.
- "Cosa Vedere": 4 rows each with name input, image URL input, description textarea (2 rows).
- "Curiosità": 3 rows each with icon input, title input, text textarea.
- Save button "Salva Destinazione" (gold primary).
- Show "Salvato con successo." if ?saved=1.
