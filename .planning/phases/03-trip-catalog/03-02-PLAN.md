  <how-to-verify>
Open `viaggi.php` in your local browser (or SiteGround live URL if deployed). Verify each item:

**1. Hero banner**
- Page loads showing a cinematic travel photo at ~38vh height (roughly 1/3 of viewport) — NOT full-screen
- "I Nostri Viaggi" appears in large Playfair Display serif font
- A short subline appears below the title
- No CTA button present (hero is purely visual)

**2. Sticky filter bar**
- Scroll down past the hero — the dual-row filter bar sticks to the top of the viewport below the site header (does NOT overlap the header)
- Row 1 shows: Tutti, America, Asia, Europa, Africa, Oceania, Medio Oriente (7 pills)
- Row 2 shows: Tutti + all theme tags from tags.json (road-trip, cultura, famiglia, coppia, avventura, etc.)
- On mobile/narrow viewport: both rows scroll horizontally without wrapping
- Active pills (initially "Tutti" on both rows) show navy #000744 background; inactive pills show transparent with white border

**3. Filter interaction — continent**
- Click "America" on Row 1 → only American trips show in the grid; count updates with a brief fade
- Click "America" again → deselects it, all trips reappear
- Click "Asia" → only Asian trips show (or empty state if none exist)

**4. Filter interaction — tags (AND logic)**
- Click "famiglia" on Row 2 → count updates to show trips with that tag
- While "famiglia" is active, click "road-trip" → only trips with BOTH tags show (AND logic)
- Click "Tutti" on Row 2 → all tag filters clear, full list returns

**5. URL deep-linking**
- With "America" filter active, check the browser URL bar → should show `?continent=america`
- Navigate to: `viaggi.php?continent=america&tag=famiglia` → page should load with those filters pre-applied (America pill active on Row 1, famiglia pill active on Row 2)

**6. Empty state**
- Apply a combination of filters that no trip matches (e.g., "Asia" continent + "road-trip" tag if no Asian road-trip exists)
- Trip grid disappears; empty state appears with the Italian message "Nessun viaggio trovato per i tuoi filtri..."
- If TALLY_CATALOG_URL is empty in config.php: a WhatsApp fallback link appears instead of an iframe
- Remove filters → trip grid reappears, empty state hides

**7. Trip cards**
- Cards in the catalog grid look identical to homepage trip cards (same photo, gradient, continent badge, status pill, title, dates, price, CTA link)
- Clicking "Scopri il viaggio" on a card navigates to `/viaggio/[slug]`

**8. Visual — brand colors**
- Active filter pill background is NAVY (#000744), NOT red or gold
- Section underlines (if any) are navy, NOT red
