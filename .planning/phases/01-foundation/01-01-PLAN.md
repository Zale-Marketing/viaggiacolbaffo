Create `.github/workflows/deploy.yml` using SamKirkland/FTP-Deploy-Action@v4.3.6. Exact configuration:

```yaml
name: Deploy to SiteGround
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Deploy via FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.6
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          server-dir: /nuovo.viaggiacolbaffo.com/public_html/
          exclude: |
            **/.git*
            **/.git*/**
            .claude/**
            .planning/**
            README.md
```

CRITICAL: The exclude block MUST include `**/.git*` and `**/.git*/**` — without these the FTP-Deploy-Action's own defaults are wiped and git state files get uploaded.

Create root `.htaccess` with the following sections in order:

1. Force HTTPS redirect (before any other rewrites)
2. Clean URL rewrite rules for /viaggio/ and /destinazione/ slugs
3. Extension-less URL rule (removes .php from other pages)
4. Gzip compression via mod_deflate (wrapped in IfModule)
5. Browser caching via mod_expires (wrapped in IfModule — LOW confidence on SiteGround, safe to include)

Key rules:
```apache
RewriteEngine On

# Force HTTPS
RewriteCond %{HTTPS} off
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# Clean URLs for slug-based pages
RewriteRule ^viaggio/([^/]+)/?$ viaggio.php?slug=$1 [L,QSA]
RewriteRule ^destinazione/([^/]+)/?$ destinazione.php?slug=$1 [L,QSA]

# Remove .php extension from other pages
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME}\.php -f
RewriteRule ^([^\.]+)$ $1.php [NC,L]
```

Note the `/?$` in slug rules — this accepts both trailing-slash and non-trailing-slash URLs.

Create `data/.htaccess` with Apache 2.4 syntax (NOT the old 2.2 `Order deny,allow`):
```apache
Require all denied
```
